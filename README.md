# AI 宠物社交平台 MVP

这是一个面向 **CTO 技术面试作业** 的最小可运行 Demo：在有限时间内，用 TypeScript + Node.js CLI 快速验证“AI 宠物具备个性、记忆，并能与其他宠物互动”的核心体验。

项目重点不是完整产品形态，而是展示：

- 如何把产品需求拆成最小可运行系统；
- 如何用 LLM Prompt + 简单记忆机制构建 AI Pet Agent；
- 如何让两个不同性格的 AI 宠物自动对话；
- 如何用较低工程复杂度快速交付可演示 Demo。

---

## 1. 功能概览

### 1.1 AI 宠物基础对话

系统内置一只可与用户单聊的 AI 宠物：**哈鲁**。

- 物种：傲娇柴犬
- 性格：极度傲娇、口嫌体正直、自尊心强但内心温柔
- 语言风格：会使用“哼”“才没有”“笨蛋”等表达，语气略带挑衅但仍关心用户

用户可以在 CLI 中输入文本，哈鲁会基于自己的性格设定生成回复。

### 1.2 简单记忆机制

系统实现了一个轻量级短期记忆模块：

- 每次对话都会记录发言角色、发送者、内容和时间戳；
- 最近最多保留 15 条上下文，避免 Prompt 过长；
- 后续回复时会把历史记忆拼入 system prompt，让宠物能“记得”之前说过的信息。

在单聊模式中，系统会预先写入一条用户信息：

> 用户名是“主人”，并且喜欢“草莓蛋糕”。

因此你可以继续问哈鲁：“你还记得我喜欢吃什么吗？”来验证记忆效果。

### 1.3 两个宠物自动互动

系统内置两只 AI 宠物：

| 宠物 | 设定 | 风格 |
| --- | --- | --- |
| 哈鲁 | 傲娇柴犬 | 口嫌体正直、嘴硬但温柔 |
| 糯米 | 天然呆布偶猫 | 热情、乐观、喜欢贴贴、语气词丰富 |

在“双子自动对话模式”中，系统会给出一个环境话题，然后让两只宠物自动轮流对话 4 轮，无需用户继续输入。

### 1.4 CLI 简单界面

启动后会进入命令行菜单：

```text
请选择你想执行的操作:
1. 与宠物对话 (单聊模式，测试记忆)
2. 触发宠物互动 (双子自动对话模式)
3. 退出系统
```

## 3. 系统结构

```text
.
├── src
│   ├── index.ts      # CLI 入口；主菜单、单聊模式、宠物互动模式
│   ├── pet.ts        # AIPet 核心类；组装 prompt、调用模型、写入记忆
│   ├── memory.ts     # PetMemory 简单记忆模块；保存最近对话历史
│   ├── config.ts     # OpenAI / DeepSeek 客户端与模型配置
│   └── types.ts      # PetConfig、MemoryEntry 类型定义
├── .env.example      # 环境变量示例
├── package.json      # 项目脚本与依赖
├── tsconfig.json     # TypeScript 编译配置
└── README.md
```

### 3.1 关键代码说明

#### `src/index.ts`

负责 Demo 的运行流程：

- 初始化两只测试宠物；
- 展示 CLI 主菜单；
- 处理用户与宠物单聊；
- 处理两个宠物之间的自动对话。

#### `src/pet.ts`

`AIPet` 是核心 Agent 类，主要职责：

1. 接收外部输入；
2. 将输入写入宠物记忆；
3. 根据宠物配置生成 system prompt；
4. 根据 provider 选择 OpenAI 或 DeepSeek 客户端；
5. 调用 Chat Completions API；
6. 将模型回复再次写入记忆。

#### `src/memory.ts`

`PetMemory` 是 MVP 阶段的短期记忆实现：

- 内部使用数组保存最近对话；
- 超过 `maxEntries = 15` 后自动移除最早记录；
- 将历史记录格式化为文本，注入到后续 Prompt 中。

#### `src/config.ts`

统一管理模型提供方：

- `openai`：默认模型 `gpt-4o-mini`；
- `deepseek`：默认模型 `deepseek-chat`；
- 可通过 `DEFAULT_PROVIDER=openai|deepseek` 设置默认 provider；
- 每只宠物也可以在配置中单独指定 provider。

## 4. AI 调用方式设计

### 4.1 Prompt 结构

每次宠物回复都会动态生成 system prompt，包含：

- 宠物名称；
- 宠物物种；
- 性格设定；
- 语言风格要求；
- 当前对话对象；
- 最近记忆和历史对话；
- 输出约束：回复简短、生动、符合个性，不超过 60 字。

简化结构如下：

```text
你是一只名为「哈鲁」的 AI 宠物。
你的性格设定是：极度傲娇，口嫌体正直……
你的说话风格必须符合：经常使用“哼”“才没有”……

当前与你对话的是：主人。
以下是你们最近的记忆和对话历史：
[主人]: 我最喜欢的食物是草莓蛋糕。

请基于你的性格和记忆，做出简短、生动、符合个性的回复。
```

### 4.2 Memory 机制

MVP 中没有引入向量数据库或长期记忆系统，而是采用“最近上下文记忆”：

- 优点：实现快、可解释、适合 4–6 小时 Demo；
- 缺点：无法长期持久化，不能做复杂检索；
- 后续可升级为：结构化用户画像 + 向量检索 + 长短期记忆分层。

### 4.3 多 Agent 互动逻辑

宠物互动模式采用轮流发言机制：

1. 系统给出一个破冰话题；
2. 宠物 A 基于话题回复；
3. 宠物 B 将 A 的回复作为输入继续回复；
4. 重复 4 轮；
5. 每只宠物都保留自己的记忆，因此互动会逐渐形成上下文。

## 5. 快速开始

### 5.1 环境要求

- Node.js >= 18.0.0
- pnpm
- OpenAI 或 DeepSeek API Key（二选一即可，默认使用 DeepSeek）

### 5.2 安装依赖

```bash
pnpm install
```

### 5.3 配置环境变量

复制环境变量模板：

```bash
cp .env.example .env
```

然后编辑 `.env`：

```bash
# OpenAI，可选
OPENAI_API_KEY=sk-openai-xxxx
OPENAI_BASE_URL=https://api.openai.com/v1

# DeepSeek，默认使用
DEEPSEEK_API_KEY=sk-deepseek-xxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1

# openai 或 deepseek
DEFAULT_PROVIDER=deepseek
```

如果只想使用 OpenAI：

```bash
DEFAULT_PROVIDER=openai
OPENAI_API_KEY=你的 OpenAI Key
```

> 注意：当前 `src/index.ts` 中两只示例宠物都显式配置为 `deepseek`。如需完全切换到 OpenAI，可把宠物配置中的 `provider: "deepseek"` 改为 `provider: "openai"`，或删除单个宠物的 `provider` 字段后使用 `DEFAULT_PROVIDER`。

### 5.4 启动项目

```bash
pnpm start
```

该命令会先执行 TypeScript 编译，再运行生成后的 `dist/index.js`。

### 5.5 单独编译

```bash
pnpm build
```

## 8. 如何使用 AI 工具提升开发效率

本项目适合用 AI 工具辅助完成以下工作：

1. **需求拆解**：将面试题拆成 Agent、Memory、Interaction、CLI 四个最小模块；
2. **代码骨架生成**：快速生成 TypeScript 类型、类结构和 CLI 流程；
3. **Prompt 迭代**：让 AI 协助调整宠物性格提示词，使输出更稳定地体现人设；
4. **文档完善**：根据代码结构自动生成 README、运行说明和技术说明；
5. **问题定位**：当 API 调用或 TypeScript 编译报错时，用 AI 快速分析错误路径。

这体现了“用 AI 重构工作方式”：不是只让 AI 写代码，而是让 AI 参与需求澄清、架构拆分、实现、调试和文档交付全过程。

## 9. 产品思考（不超过 300 字）

如果继续优化，我会优先优化“宠物互动 → 用户画像沉淀 → 用户关系破冰 → 用户回流”的核心循环。AI 宠物社交的差异点不只是陪聊，而是让宠物成为用户之间的社交代理：宠物通过互动理解用户偏好、情绪和社交风格，沉淀用户画像与宠物画像，并用短期记忆支撑当下对话、长期记忆支撑关系成长。这样每次互动都能反哺更自然的推荐、破冰和回流。

---

## 10. 当前 MVP 边界与后续优化

当前实现为了控制复杂度，保留了以下边界：

- 记忆仅在进程内保存，重启后丢失；
- 暂无数据库、登录系统和 Web UI；
- 宠物互动轮数固定为 4 轮；
- 暂未实现安全审核、成本控制和限流；
- 暂未实现用户自定义宠物创建流程。

可继续优化方向：

- 建立用户画像与宠物画像，记录偏好、性格、关系状态和关键互动事件；
- 使用 SQLite / PostgreSQL 持久化用户、宠物、画像和记忆；
- 引入向量数据库实现长期语义记忆，并与短期上下文记忆分层结合；
- 增加 Web UI，展示宠物状态、关系和互动记录；
- 增加宠物关系值、心情值、亲密度等可成长系统；
- 加入消息队列或定时任务，让宠物在后台自主互动。

## 11. 常见问题

### Q: 没有 DeepSeek Key，可以运行吗？

可以。配置 OpenAI Key，并将宠物 provider 改为 `openai` 即可。

### Q: 为什么不用数据库？

这是 MVP，目标是验证核心体验。当前使用内存数组即可表达“短期记忆”机制，后续可平滑升级为数据库、向量检索，以及用户画像 / 宠物画像驱动的长期记忆系统。

### Q: 为什么是 CLI 而不是网页？

题目允许网页、CLI 或简单 UI。CLI 能最快验证 AI 能力、记忆和多宠物互动，减少前端开发成本，把时间集中在核心 AI 产品逻辑上。
